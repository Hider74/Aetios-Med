üöÄ 3. Installation Guide

Installation Methods

There are two ways to install Aetios-Med:

  1. Pre-built Application - Download and install (easier, recommended for most users)
  2. Build from Source - Clone and build yourself (for developers or customization)

Method 1: Pre-built Application (Coming Soon)

Pre-built installers for Windows (.exe), macOS (.dmg), and Linux (.AppImage) will be available on the GitHub releases page. This manual focuses on building from source, which gives you the most up-to-date version.

Method 2: Build from Source

Step 1: Clone the Repository

Open your terminal (macOS/Linux) or Command Prompt/PowerShell (Windows) and run:

# Clone the repository
git clone https://github.com/Hider74/Aetios-Med.git

# Navigate into the directory
cd Aetios-Med

Step 2: Install Backend Dependencies

For Windows:

# Navigate to backend folder
cd backend

# Install Python dependencies
pip install -r requirements.txt

# Return to root
cd ..

For macOS/Linux:

# Navigate to backend folder
cd backend

# Install Python dependencies (use pip3 if pip doesn't work)
pip3 install -r requirements.txt

# Return to root
cd ..

üí° Virtual Environment (Recommended)

For cleaner Python dependency management, use a virtual environment:

cd backend
python -m venv venv

# Activate (Windows)
venv\Scripts\activate

# Activate (macOS/Linux)
source venv/bin/activate

pip install -r requirements.txt

Step 3: Install Frontend Dependencies

# Navigate to frontend folder
cd frontend

# Install Node.js dependencies
npm install

# Return to root
cd ..

Step 4: Install Root Dependencies

# In the root Aetios-Med directory
npm install

Step 5: Build for Production (Optional)

For macOS:

./scripts/build-mac.sh

For Windows:

powershell -ExecutionPolicy Bypass -File scripts/build-win.ps1

‚úÖ Installation Complete!

You're ready to start Aetios-Med. The first time you run it, you'll be guided through the setup wizard to download the AI model and configure your folders.

Running Aetios-Med

Development Mode (3 Terminal Windows)

For development or testing, run these commands in separate terminal windows:

Terminal 1 - Backend:

cd backend
uvicorn app.main:app --reload --port 8741

Terminal 2 - Frontend:

cd frontend
npm run dev

Terminal 3 - Electron:

npm run electron:dev

Production Mode

After building, just run the application from your applications folder or by double-clicking the executable.

‚ö†Ô∏è Common Installation Issues

  • Permission denied: On macOS/Linux, make scripts executable: chmod +x scripts/*.sh
  • Python not found: Try python3 instead of python
  • npm install fails: Clear cache with npm cache clean --force and try again
  • Port 8741 in use: Close other applications using this port or change it in backend/app/config.py

üé¥ 6. Anki Integration

What is Anki?

Anki is a popular flashcard application that uses spaced repetition. Many medical students use Anki for memorization. Aetios-Med integrates with Anki to:

  • Import your existing Anki decks
  • Map cards to curriculum topics automatically
  • Track which cards are related to weak topics
  • Predict when you'll forget information (memory decay)
  • Generate targeted study recommendations

Exporting Decks from Anki

Step 1: Open Anki

Launch Anki on your computer (download from apps.ankiweb.net if you don't have it).

Step 2: Select a Deck

Click on the deck you want to export (e.g., "Medicine Year 2" or "Cardiology").

Step 3: Export

  1. Go to File ‚Üí Export
  2. Choose "Anki Deck Package (*.apkg)" as the export format
  3. Check "Include scheduling information" (important for retention tracking)
  4. Check "Include media" if your cards have images
  5. Click Export and save the .apkg file

üí° Export Tips

  • Export individual decks rather than "All Decks" for better organization
  • The scheduling information is crucial for Aetios-Med to understand which cards you struggle with
  • .apkg files are just zip files containing a SQLite database

Importing into Aetios-Med

Method 1: Automatic Import (Recommended)

  1. Save your .apkg files to the Anki folder you configured during setup
  2. Aetios-Med will automatically detect and import new files
  3. You'll see a notification when import is complete

Method 2: Manual Import

  1. Go to Dashboard ‚Üí Import
  2. Click "Import Anki Deck"
  3. Browse to your .apkg file
  4. Choose whether to enable automatic topic mapping (recommended)
  5. Click Import
üì∏ Anki Import Screen

File selection dialog with import options

Automatic Topic Mapping

Aetios-Med uses AI to map your Anki cards to curriculum topics automatically:

  1. 1 The system reads the front and back of each card
  2. 2 It analyzes the content using the embedding model
  3. 3 Cards are matched to the most relevant curriculum topic(s)
  4. 4 You can review and adjust mappings manually

Example Mappings:

Card Front Mapped Topic Confidence
"What are the 4 chambers of the heart?" Cardiovascular Anatomy 95%
"First-line treatment for hypertension?" Cardiovascular Pharmacology 88%
"Symptoms of diabetic ketoacidosis" Endocrine Disorders 92%

Viewing Anki Statistics

Once imported, you can view detailed statistics about your Anki cards:

  • Total cards in each deck
  • Cards reviewed vs. new cards
  • Average ease factor (how difficult cards are for you)
  • Total lapses (times you forgot cards)
  • Weak cards (ease factor < 2.0 or multiple lapses)
  • Memory stability predictions
üì∏ Anki Statistics Dashboard

Graphs and charts showing card performance by topic

Understanding Retention Tracking

Aetios-Med tracks your Anki card retention using the card's scheduling information:

  • Interval - Days until next review
  • Ease factor - How easy/hard the card is (2.5 is normal)
  • Reviews - Number of times you've seen the card
  • Lapses - Number of times you've forgotten it
  • Last review - When you last studied this card

This data helps the AI tutor identify which topics need more attention.

Managing Multiple Decks

You can import multiple Anki decks:

  1. Each deck is tracked separately
  2. View per-deck statistics in the Anki panel
  3. Filter the knowledge graph by deck to see only relevant topics
  4. Delete imported decks if you no longer need them

‚ö†Ô∏è Important Notes

  • Importing a deck doesn't modify your original Anki database
  • Changes in Aetios-Med don't sync back to Anki (it's one-way import)
  • Re-import a deck to update statistics after studying in Anki
  • Large decks (1000+ cards) may take a few minutes to process

Understanding Topic Relationships

Prerequisites (Incoming Edges):

Topics you should understand before studying this one. For example:

  • Cardiac Pathology requires Cardiac Anatomy and Cardiac Physiology
  • Pharmacokinetics requires Biochemistry Basics

Dependencies (Outgoing Edges):

Topics that build on this one. For example:

  • Anatomy is prerequisite for Physiology, Pathology, and Clinical Skills

Tracking Confidence Over Time

Your confidence in each topic changes based on:

  • Quiz performance - Correct answers increase confidence
  • Study sessions - Time spent studying increases confidence
  • Memory decay - Confidence naturally decreases over time (forgetting curve)
  • Manual updates - You can adjust confidence yourself

Identifying Weak Areas

  1. 1 Filter by red topics - Shows urgent review needs
  2. 2 Check prerequisites - Red topics with red prerequisites need foundational work first
  3. 3 Sort by decay - Find topics you studied but are forgetting

üí° Study Strategy Tips

  • Clustered learning: Study related topics together (e.g., all cardiovascular topics)
  • Maintenance mode: Even green topics need occasional review to prevent decay
  • Exam focus: Prioritize topics directly tested in upcoming exams

Adaptive Quiz Difficulty

How Adaptive Difficulty Works:

When you generate a quiz, Aetios checks your confidence level for the topic and automatically selects appropriate question difficulty:

Confidence Level Node Color Auto-Selected Difficulty Learning Goal
Low (0-30%) üî¥ Red Easy questions Build foundational knowledge, boost confidence

üéØ Why Adaptive Difficulty Matters

  • Red topics (Low confidence): Starting with hard questions would be discouraging. Easy questions build confidence and reinforce basics.
  • Amber topics (Medium confidence): Mixed difficulty helps you apply knowledge and identify remaining gaps.
  • Green topics (High confidence): Hard questions prevent complacency and ensure you can handle clinical scenarios.

Understanding FSRS-4 Spaced Repetition

FSRS-4 is a modern spaced repetition algorithm that's more accurate than traditional SM-2 (used by Anki).

How It Works:

  1. 1 Initial Learning: New topics start with short review intervals
  2. 2 Success Increases Interval: Correct answers increase time until next review
  3. 3 Failure Resets: Incorrect answers shorten the review interval
  4. 4 Difficulty Adjustment: Algorithm learns which topics are harder for you
  5. 5 Optimal Timing: Reviews scheduled just before you're likely to forget

Review Schedule Example:

Review # Interval If Correct If Incorrect
1 1 day Next: 3 days Next: 10 minutes
2 3 days Next: 8 days Next: 1 day
3 8 days Next: 21 days Next: 3 days
4 21 days Next: 45 days Next: 8 days

  • Weak Areas: Topics where you consistently struggle

üîß 12. Troubleshooting & FAQ

Common Installation Issues

Problem: "Node not found" or "Python not found"

Solution:

  • Verify installation: node --version and python --version
  • On Windows, check "Add to PATH" was selected during installation
  • On macOS/Linux, try python3 instead of python
  • Restart your terminal/command prompt after installation
  • On macOS, install via Homebrew: brew install node python@3.10

Problem: "Permission denied" when running scripts

Solution:

  • On macOS/Linux: chmod +x scripts/*.sh
  • On Windows: Run PowerShell as Administrator
  • Or use: bash scripts/build-mac.sh instead of ./scripts/build-mac.sh

Problem: npm install fails with EACCES

Solution:

  • Don't use sudo with npm
  • Fix npm permissions: npm docs
  • Or use npm install --prefix ~/my-npm-packages

Problem: pip install fails

Solution:

  • Use a virtual environment (recommended)
  • On macOS/Linux: pip3 install --user -r requirements.txt
  • Update pip: python -m pip install --upgrade pip
  • Check Python version is 3.10+ with python --version

Backend Connection Problems

Problem: "Cannot connect to backend" error

Solution:

  1. Check backend is running: Visit http://localhost:8741/docs
  2. If not running, start it: cd backend && uvicorn app.main:app --reload --port 8741
  3. Check firewall isn't blocking port 8741
  4. Check another application isn't using port 8741: lsof -i :8741 (macOS/Linux) or netstat -ano | findstr :8741 (Windows)
  5. Try a different port in backend/app/config.py

Problem: Backend starts but crashes immediately

Solution:

  • Check Python dependencies are installed: pip list
  • Look for error messages in the backend terminal
  • Common issues:
    • Missing database: The app will create it automatically on first run
    • Missing ChromaDB: pip install chromadb
    • CUDA errors: Set GPU layers to 0 in settings (CPU-only mode)

Model Download Issues

Problem: Model download is very slow

Solution:

  • Use a stable internet connection (avoid mobile hotspots)
  • Download can be paused and resumed
  • Try downloading during off-peak hours
  • Alternatively, manually download from HuggingFace and place in models folder

Problem: Model download fails or gets corrupted

Solution:

  • Delete partially downloaded file from models folder
  • Try again with a HuggingFace token (faster servers)
  • Check available disk space (need ~5GB free)
  • Disable VPN if you're using one

Problem: "Model not found" even after download

Solution:

  • Check model is in correct location:
    • Windows: %APPDATA%\Aetios-Med\models\llama3-openbio-8b.Q4_K_M.gguf
    • macOS: ~/Library/Application Support/Aetios-Med/models/llama3-openbio-8b.Q4_K_M.gguf
    • Linux: ~/.local/share/Aetios-Med/models/llama3-openbio-8b.Q4_K_M.gguf
  • Check file size is ~4.5GB (not 0 bytes)
  • Restart the application

Performance Issues

Problem: AI responses are very slow (>60 seconds)

Solution:

  • Close other applications to free RAM
  • Check CPU usage - if 100%, close background apps
  • On laptops, plug in power (performance mode)
  • First response is slower (model loading) - subsequent responses faster
  • If consistently slow, your hardware may be below minimum requirements

Problem: App is laggy or unresponsive

Solution:

  • Clear browser cache in Settings
  • Reduce number of open tabs/windows
  • Check RAM usage - close if using > 90%
  • On knowledge graph, use fewer nodes (filter by topic)
  • Restart the application

Problem: GPU acceleration not working

Solution:

  • NVIDIA: Install CUDA toolkit and drivers
  • Apple Silicon: Should work automatically with Metal
  • Check Settings ‚Üí Model ‚Üí GPU Layers (set to -1 for all)
  • If problems persist, set to 0 (CPU only)

Anki Integration Issues

Problem: .apkg file won't import

Solution:

  • Ensure file has .apkg extension (not .zip)
  • Try exporting from Anki again with "Include scheduling information" checked
  • Check file isn't corrupted: Open in Anki first to verify
  • Large decks (5000+ cards) may take several minutes

Problem: Cards not mapping to topics

Solution:

  • Enable "Automatic topic mapping" during import
  • Very specialized cards may not map well (manual mapping available)
  • Check embedding model is downloaded (happens automatically)

Data & Storage Issues

Problem: "Database locked" error

Solution:

  • Close all Aetios-Med instances
  • Wait 30 seconds
  • Restart the application
  • If persistent, delete aetios.db-shm and aetios.db-wal files (not aetios.db)

Problem: Running out of disk space

Solution:

  • Model: ~4.5GB (required)
  • Database: Grows with usage (~100-500MB typical)
  • Vector store: ~1-2GB with many documents
  • Can delete old imported Anki decks if no longer needed
  • Can clear chat history (Settings ‚Üí Privacy)

Q: How often should I update my confidence levels?

UK Medical Education Context

Typical Medical School Structure:

  • Years 1-2: Pre-clinical - Basic sciences (anatomy, physiology, pathology)
  • Years 3-5: Clinical rotations - Hospital placements, patient contact
  • Year 5: Finals exams - Written (UKMLA) and clinical (OSCEs)
  • Foundation Years: F1/F2 - Supervised clinical practice in NHS

Key Exams:

  • UKMLA (Applied Knowledge Test): Multiple-choice exam testing clinical knowledge
  • UKMLA (Clinical): OSCE-style clinical skills assessment
  • Finals: Medical school's own assessments (may include UKMLA)
  • OSCEs: Objective Structured Clinical Examinations - stations testing clinical skills
